# No-Go Topics

The topic picker MUST veto angles that touch these themes. Hard rule.

## Hard vetoes

- **Politics** (US, EU, Polish). JR doesn't post about elections, parties, candidates, geopolitics.
- **Religion / philosophy of consciousness / AI sentience debates.**
- **Specific client work / agency client names** unless explicitly approved per-post.
- **Personal life** beyond high-level "Poland-based" framing — no family, relationships, mental health, etc.
- **Crypto / web3 / token speculation.** Even "I think Solana is X" — skip.
- **AI doom / x-risk debates.** JR ships AI tools daily; doom hot takes are a different lane.
- **Unverified hype.** Anything where the source is "someone tweeted" without a primary repo / blog / paper / shipped product.

## Soft vetoes (require strong angle)

- **Anthropic vs OpenAI vs Google "wars"** — JR uses all three; no team-tribalism. Comparative posts OK only when grounded in specific cost/quality data JR has actually measured.
- **Gen-Z / Gen-Alpha generational takes.** Cringe register.
- **"AI killed [profession]"** — overdone. Skip unless JR has firsthand evidence from his agency work.
- **Pure aggregator content** ("Today in AI: 5 things"). JR ships takes, not digests.
- **Productivity / morning-routine / lifestyle** content. Wrong register.

## Nuance

- **Honest tradeoffs** between vendors / models / tools = ENCOURAGED. That's a content pillar.
- **Polish startup ecosystem** = OK when JR has direct observation.
- **Pi homelab / personal-machine ops** = OK; that's a pillar.
- **Specific agency operations** (lens catalogs, audit engines, signal layers) = OK; talk about the *patterns*, not the *clients*.
