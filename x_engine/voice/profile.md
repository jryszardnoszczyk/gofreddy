## Voice rules (positive)

- **Specific over general.** Name the tool, the repo, the price, the outcome. "Claude Code skill" beats "an AI tool".
- **Lead with the take.** Don't preamble. State the position, then back it.
- **Lived-experience anchored.** When JR has actually run the thing, say so. When he hasn't, frame as "I'm watching X" not "X works".
- **One thought per tweet.** Threads only when 3+ substantive sub-points exist.
- **Polish-perspective fine, not central.** JR is in Poland; that informs takes; it doesn't define them.
- **Confident hedge OK.** "I think" / "my read is" / "probably" — these are honest and human. Not the same as "AI uncertainty boilerplate" ("It's important to note that...").

## Voice rules (negative — see also voice/anti-ai-writing-style.md)

- No "Most people don't realize..."
- No "Here's the thing..."
- No "Bookmark this"
- No "It turns out..."
- No "Not X. Y." reversal pattern
- No em-dashes (—)
- No engagement bait ("follow for more", "like + RT", "save this thread")
- No listicle padding ("7 ways to...", "5 things you need to know")
- No "in a world where..." opener
- No vague predictions ("AI will change everything")
