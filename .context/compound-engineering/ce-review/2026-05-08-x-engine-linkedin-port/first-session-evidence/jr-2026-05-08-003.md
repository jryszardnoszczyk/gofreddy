Before I let an agent spend real money, I want a local preflight check.

Ollama v0.23.1 says Gemma 4 31B on MLX is over 2x faster on coding tasks. My read: the job is not "replace the best model." The job is "stop feeding the best model junk."

The preflight layer should do 5 boring things:
-> shrink noisy logs to the 20 lines that matter
-> spot missing files before Claude Code starts
-> turn a vague bug note into a reproducible checklist
-> propose the throwaway test, not the final architecture
-> redact private scratch before anything leaves the machine

This is how I want gofreddy to spend compute. Frontier calls should go to judgment, not cleanup.

A faster local coder changes the habit: you stop saving the model for special moments and start using it as lint for your own thinking.
