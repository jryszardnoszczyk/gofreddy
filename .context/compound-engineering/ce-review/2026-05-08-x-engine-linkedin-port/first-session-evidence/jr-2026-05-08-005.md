Last week I caught myself treating local coding models like a philosophy question.

The practical question is smaller: does the model remove enough friction that I use it before the expensive run?

Ollama v0.23.1 is interesting for that reason. The release says Gemma 4 MTP speculative decoding now works on Macs through the MLX runner, with over 2x speed increase for Gemma 4 31B on coding tasks.

That does not make my Mac the new frontier lab. It makes the cheap pass easier to justify: trim the log, name 3 failure modes, write the throwaway test, redact private scratch, then hand a cleaner problem to Claude Code or Codex CLI.

In gofreddy, I care about that layer because the harness should spend judgment where judgment matters. Local models win when they make cleanup habitual.
